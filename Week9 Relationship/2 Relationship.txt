1 - First Step config 

@OneToOne(fetch = FetchType.EAGER,cascade = CascadeType.PERSIST, targetEntity = Roles.class) 

Explain: Parent to child 

1. FetchType.EAGER # FetchType.Lazy : we think about it when starting the system it will get the data or no 

in default One to One is explicit:

- By default all ToMany relationship are Lazy ព្រោះបើវា load data មកទាញ ទិន្នន័យម្តងវាស៊ីធំ។
- អីក៏ដោយ ToOne relationship are Eager. ទិន្នន័យតិចវា Load data មកតែម្តង់

2. cascade = CascadeType.PERSIST : បើ Parent ធ្វើអី កូននឹងវាធ្វើនឹងដូចគ្នា Ex: if we delete the person, 
the child is also deletd that person id . 

3. targetEntity : ភ្ជាប់ទៅ Entity មួយណា​ ចឹងយើងត្រូវដាក់ឈ្មោះ Class នឹង

4. ចុះយើងធ្វើម្តេចដឹងថាវាជា unibirectional :

2. Config on JoinColumn

@JoinColumn(name = "role_id", referencedColumnName = "roleId",nullable = false)

- name = "role_id" : Defines the column name for the foreign key in the current table. តើវាភ្ជាប់ទៅ Entity មួយណា

- referencedColumnName = "roleId" : Specifies that role_id refers to the roleId column in the Roles table.

- nullable = false : អត់មានក៏បានដែល
